# Checklist de Auditoría Estática – IEEE 1028
Proyecto: DemoAuditApp
Objetivo: Evaluar calidad del código sin ejecutar el sistema

---

## 1. Estándares de Código
- Uso correcto de nombres de clases y métodos
- Eliminación de código muerto
- Uso consistente de llaves
- Evitar valores mágicos
- Código legible y ordenado

---

## 2. Seguridad Básica
- Validación de entradas de usuario
- Manejo de excepciones
- Prevención de errores por parsing
- No exposición de información sensible

---

## 3. Mantenibilidad
- Complejidad baja
- Métodos pequeños
- Código modular
- Fácil modificación futura

---

## 4. Documentación
- Comentarios XML en métodos
- Explicación de lógica crítica
- Código autoexplicativo

---

## 5. Buenas Prácticas C#
- Uso de TryParse
- Uso de try-catch
- Manejo correcto de flujos

---

Resultado:
Detección temprana de defectos antes del despliegue
